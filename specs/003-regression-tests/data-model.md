# Data Model: Regression Test Coverage

**Date**: 2026-07-11 | **Plan**: [plan.md](plan.md)

No persistent or wire data changes. The entities are test artifacts and test-double state.

## Reference corpus (`sim/test/vectors/reference.json`) ‚Äî consumed by G1

Committed ground truth, generated by `vectorgen`; schema owned by `specs/001-web-simulator/contracts/reference-vectors.md`. Relevant structure for the new consumer:

| Key | Content | Comparison rule |
|-----|---------|-----------------|
| `meta.effectSeeds`, `meta.*` | Generator provenance (incl. `firmwareGitDescribe`) | Seeds are *validated* (must match what `VectorGenCommon` predicts); `firmwareGitDescribe` is **ignored** |
| `effects[]` | Wire index ‚Üî effect name table | Must match the live registry exactly (order, size, last-two invariant) |
| `palettes[]` | Palette table | Must match `Effect::palettes()` (size 22, values) |
| `cases[]` (1,380) | `{id, device, effect, palette, timeMs, leds[[r,g,b],...]}` | Every case re-rendered by `VectorGenCommon`; RGB triples byte-exact |
| `primitives` | FastLED math samples (sin8, random16 stream, ‚Ä¶) | Re-evaluated against FakeFastLED on host |

**Lifecycle rule (FR-003)**: intentional behavior changes regenerate the corpus (`make vectorgen && ./vectorgen > sim/test/vectors/reference.json`) in the same commit; the C++ test and the JS suite then both gate against the new truth.

## VectorGenCommon (new shared case model) ‚Äî G1

Extracted, not invented: the single source of truth for corpus semantics.

- `PredictEffectSeedsAndResetPrngs()` ‚Äî deterministic construction-time randomness for Fire/Firefly/Rorschach (srand(1), random16 seed 1337)
- Effect/palette wire tables; the three-device catalog (scarf/puck/ufo)
- `RenderCase(case) -> vector<CRGB>` ‚Äî the operation both the generator (emit) and the test (compare) call

**Invariant**: `vectorgen`'s emitted JSON before and after the extraction is byte-identical (verified in commit 2's gate).

## FakeRadio wire-path state ‚Äî G2

| Field | Before | After |
|-------|--------|-------|
| sent packet | `RadioPacket*` struct copy | wire buffer (`uint8_t[61]` + length) written by `Serialize` in `sendPacket`; `getSentPacket` returns `Deserialize`d packet or `nullptr` if rejected |
| received packet (injection) | raw struct pointer, memcpy in `readPacket` | **unchanged** ‚Äî invalid-packet injection bypasses the codec by design (FR-005) |

**Invariant preserved**: valid traffic is codec-transparent (all existing suites pass unchanged); a codec defect is visible to every consumer of `getSentPacket` (FakeNetwork mesh, RadioStateMachineTest, NetworkManagerTest).

## Render-loop test fixtures ‚Äî G3

- Stub effect: `color = f(led_index)` (pure, no palette, defined in test) registered via `ClearEffects()`/`PublicAddEffect()`
- Device fixtures: four-strip flag device [plain, Off, Dim, Reversed], a Dim+Reversed strip, multi-strip offset device
- Oracle: the plain strip's output; flags asserted as transformations of it

## Leaf-utility cases ‚Äî G4

- `DeviceDescription::GetLedCount`: 0 / 1 / many strips; type-ceiling boundary pinned as documented current behavior
- `ColorPalette::GetGradient`: single color, exact boundaries, wrap vs no-wrap, extremes ‚Äî table-driven
- `BatteryVoltageToRawReading`: monotonic; round-trips within one quantization step over the calibration range
- Palette registry boundary: `palettes().size()` == corpus table size; all valid indices render (OOB is UB ‚Äî documented, not executed; see research R4)
