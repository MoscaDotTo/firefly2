Config blob · proposed flash format v1
Firmware-side struct (the contract this blob implements) 
// 32 bytes, little-endian, written at 0x0001E000 on node
// (kEepromStart = 0x20000 − 0x2000). Implemented in firmware as
// DeviceDescription::SerializeV1 / FromV1 — falls back to
// Devices::current on magic/version/CRC mismatch.
struct FlashConfigV1 {
  char     magic[4];      // "FFLY"
  uint8_t  version;       // 1
  uint8_t  strip_count;   // 1..8
  uint16_t reserved;      // 0
  uint32_t milliamps_supported;
  struct { uint8_t led_count; uint8_t flags; } strips[8];
  uint32_t crc32;         // IEEE CRC-32 of bytes 0..27
};
Live preview faithful ports of the C++ effects · rendered at network time

Honors Reversed / Off / Dim exactly as LedManager::RunEffect does; Tiny / Bright / Circular / Mirrored are interpreted inside each effect, as on hardware.

Morse pattern generator select “Morse beacon” above to preview

ITU timing: dit = 1 unit, dah = 3, symbol gap = 1, letter gap = 3, word gap = 7; the loop ends with a word gap so repeats read cleanly. Blink and marquee modes are in firmware as MorseEffect (Mode::kBlink / Mode::kMarquee, 120 ms unit, palette-colored, compiled-in messages registered in LedManager.cpp). Marquee prints the pattern one unit per LED and scrolls one LED per unit; long patterns wrap. Freeform messages here and the static mode are page-side design tools; use the message string or C export to promote a pattern into firmware.

Flash · config blob → UF2

Firmware requirement: the board must run firmware that includes DeviceDescription::FromV1 (node builds now default to READ_FROM_FLASH). Flash the firmware once via the converter below; after that, config-only flashes are all you need. Older firmware ignores the config blob and falls back to its compiled-in device.

“Save” downloads the file for drag-and-drop onto the FIREFLY boot drive (double-tap reset to enter the bootloader). “Flash to drive” writes it directly via the File System Access API — Chromium-only; the download path works everywhere.

Flash · firmware .bin → UF2 works with current firmware

Reproduces tools/create_uf2.py in the browser: family 0x68ED2B88 (SAMD21), base 0x2000. Feed it .pio/build/node/firmware.bin.
